# system-architecture-v2.md

## 핵심 원칙
식단메이트는 말은 친구처럼 하지만, 속은 엔진처럼 돌아가야 한다.

LLM 하나로 모든 것을 해결하려 하면 안 된다.
대화와 계산을 분리해야 신뢰성과 확장성이 나온다.

---

## 시스템 구성

### 1. Conversation / Voice Agent
역할:
- 음성 대화
- 체크인
- 사용자 의도 파악
- 설명/설득/유도
- 요리 모드 진행

기술 후보:
- OpenAI Realtime API
- Whisper/STT
- TTS 출력

---

### 2. Planner Engine
역할:
- 일간/주간/월간 식단 생성
- 고정/교체/재생성 반영
- 다양성 제약조건 반영
- 외식/현실 변수 반영

예시 규칙:
- 생선 주 2회 이상
- 같은 메인 단백질 2일 연속 금지
- 같은 레시피 14일 내 재등장 금지
- 나트륨 높은 메뉴 주간 상한 적용

---

### 3. Nutrition Calculator
역할:
- 레시피 → 영양소 합산
- KDRI 기반 권장 패턴 계산
- 일 단위/주 단위 목표 분리

데이터 소스 방향:
- 식약처 식품영양성분 DB
- 농촌진흥청 국가표준식품성분 DB
- 전국통합식품영양성분정보 표준데이터

---

### 4. Price Engine
역할:
- 주간 장보기 예산 추정
- 대체재 기반 비용 최적화
- 온라인/공공가격 기반 가성비 추천

데이터 방향:
- KAMIS
- 공공 가격 데이터
- 쇼핑 검색 API

---

### 5. Store Finder
역할:
- 주소/반경 기반 구매처 추천
- 마트/시장/정육점/채소가게 후보 탐색
- 품목별 구매 전략 제안

데이터 방향:
- 네이버 지역 검색 API
- 지도/POI 검색

---

### 6. Memory
역할:
- 좋아하는 메뉴
- 싫어하는 음식
- 회피 패턴
- 성공률 높은 식단
- 예산 선호
- 외식 빈도

메모리는 장기 개인화의 핵심이다.

---

### 7. Feedback Loop
역할:
- 실제 실행 결과 수집
- 수행률 계산
- 복귀율 계산
- 다음 식단/다음 주 계획 자동 보정

핵심:
식단메이트는 고정 계획 앱이 아니라,
실행 결과로 계속 업데이트되는 운영 시스템이어야 한다.
