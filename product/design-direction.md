# design-direction.md

## 제품 성격
식단메이트는 앱스토어에 올라갈 **소비자용 모바일 앱**이다.
따라서 랜딩/소개 페이지와 앱 UI 모두 아래 성격을 가져야 한다.

- 건강하지만 과하게 의료적이지 않음
- 친근하지만 유치하지 않음
- 예쁘지만 복잡하지 않음
- 실행 중심, 저마찰, 생활 밀착형

## 디자인 목표
1. 첫인상에서 "부담 없는 식단 앱"처럼 보여야 한다.
2. 기존 칼로리 기록 앱보다 훨씬 가볍고 쉬워 보여야 한다.
3. 웰니스/라이프스타일 앱처럼 부드럽지만, 실제로는 실용적인 도구처럼 보여야 한다.
4. 앱스토어 스크린샷으로 잘 잘려도 메시지가 살아야 한다.

## 추천 톤
### 방향
- Warm Wellness + Clean Utility

### 키워드
- soft
- fresh
- low-friction
- modern
- calm
- friendly

### 피해야 할 톤
- 병원/의료 서비스처럼 차가운 톤
- 다이어트 강박/죄책감 자극 톤
- 지나치게 여성향 감성 앱 느낌
- 피트니스/근성 앱 느낌

## 컬러 방향
### 메인
- Warm Green / Mint / Soft Lime 계열
- 깨끗한 White 또는 Warm Gray 배경

### 보조
- Peach / Soft Yellow / Calm Beige 포인트 가능

### 원칙
- 식욕을 자극하기보다 신선함과 가벼움을 줘야 함
- 건강 관리 앱처럼 보이되 병원 앱처럼 보이면 안 됨

## 타이포 방향
- 제목: 간결하고 힘 있게
- 본문: 짧고 편안하게 읽히게
- 숫자/지표는 강조 가능하되 부담스럽지 않게

## UI 방향
### 홈
- 오늘의 식단 추천이 가장 먼저 보여야 함
- 복잡한 데이터보다 "지금 뭘 할지"가 보이게

### 기록
- 사진 + 빠른 선택 버튼 중심
- 수치 입력은 최소화

### 복귀 UX
- 부드럽고 비판 없는 문구
- "괜찮아요, 다음 한 끼부터 다시 가요" 같은 방향

## 랜딩페이지 방향
### 목적
- 앱스토어 올릴 제품의 초기 소개 페이지
- 베타 모집 / 문제 공감 / 가치 전달

### 섹션 제안
1. Hero
2. 문제 공감
3. 핵심 가치 3개
4. 앱 사용 흐름
5. 스크린샷/목업 영역
6. 베타 CTA

## 핵심 카피 톤
- 오늘 뭐 먹지?
- 망쳐도 괜찮아요. 다음 한 끼부터 다시.
- 칼로리 앱처럼 귀찮지 않게.
- 식단을 잘 짜주는 앱보다, 식단을 계속하게 해주는 앱.

## 앱스토어 고려사항
- 스크린샷 한 장당 메시지 1개
- 첫 3장 안에 핵심 가치 설명
- 아이콘/브랜드 톤은 친근하지만 너무 장난스럽지 않게
- 웰니스 카테고리 경쟁 앱과 비교해도 촌스럽지 않아야 함
