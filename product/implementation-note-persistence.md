# implementation-note-persistence.md

## 결정
식단메이트 MVP는 앱 재실행 시에도
- 사용자 목표/스타일/현실 조건
- 최근 기록 데이터
가 유지되도록 로컬 영속성을 붙인다.

## 이유
- MVP라도 사용자가 앱을 다시 열었을 때 상태가 초기화되면 제품 감각이 크게 떨어진다.
- 온보딩 반복은 이탈 요인이 된다.
- 주간 요약은 누적 데이터가 핵심이므로 저장 계층이 필요하다.

## 현재 선택
- AsyncStorage 기반 로컬 저장
- 서버 동기화 전 단계의 가벼운 persistence
- 추후 backend/supabase 연동 전까지 충분한 구조
