# information-architecture-v2.md

## 핵심 구조
식단메이트의 사용자 경험은 3개의 모드로 구성된다.

1. 대화 모드
2. 플래너 모드
3. 요리 모드

---

## 1. 대화 모드 (기본 진입점)
### 목적
사용자의 하루 식단 운영을 가장 자주 해결하는 기본 인터페이스

### 주요 작업
- 오늘 뭐 먹지 결정
- 외식/야식/변수 대응
- 식단 교체/대체
- 대충 기록
- 다음 끼니 자동 보정

### 기본 루프
체크인 → 오늘 식단 제안 → 수정/교체 → 기록 → 자동 보정

### 입력 방식
- 음성 우선
- 보조적으로 버튼/짧은 텍스트

---

## 2. 플래너 모드
### 목적
식단메이트의 유료 핵심 가치

### 구조
- 무료: 주간 플랜 일부 또는 제한적 생성
- 유료: 월간(4주) 식단을 내부적으로 한 번에 생성
- 실제 UI 제공은 주차별로 펼쳐서 과부하 방지

### 핵심 기능
- 주간 식단 자동 생성
- 월간 4주 식단 생성
- 식단 고정
- 식단 교체
- 식단 재생성
- 주간 장보기 리스트 연결

### 설계 원칙
- 월간은 계획 단위
- 실행은 주차별/일별 단위
- 사용자는 언제든 고정/교체/재생성 가능

---

## 3. 요리 모드
### 목적
추천이나 계획이 실제 실행으로 이어지게 만드는 모드

### 핵심 기능
- 단계별 레시피 음성 안내
- 다음/반복/타이머 명령
- 재료 부족 시 대체안 제시
- 조리 중 분량 변경 반영

### 대표 음성 명령
- 다음
- 반복
- 타이머 5분
- 불 줄여
- 1인분을 2인분으로 바꿔

---

## 보조 구조
### 기록
- 대화 모드 안에서 매우 가볍게 처리
- 별도 복잡한 기록 UX 지양

### 요약/리포트
- 주간 수행률
- 복귀율
- 다양성/균형 리포트
- 다음 주 자동 보정

### 장보기
- 플래너 모드와 강하게 연결
- 주간 장보기 중심
- 위치/가격 모드는 후속 확장
