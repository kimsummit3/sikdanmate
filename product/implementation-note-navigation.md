# implementation-note-navigation.md

## 결정
식단메이트 MVP 프로토타입은 초기 수동 화면 분기에서 시작했지만,
이후 화면 증가와 상태 확장을 고려해 React Navigation 기반 구조로 전환한다.

## 이유
- 화면 수 증가 대응
- 실제 앱 구조와 유사한 개발 방식 확보
- 이후 deep link / stack / tab 구조 확장 용이
- 화면별 책임 분리 강화

## 현재 판단
- 온보딩 / 홈 / 로그 / 요약 / 설정 화면은 이미 존재
- 지금 시점에서 내비게이션 계층을 넣어도 비용이 과하지 않음
- 이후 store/state 구조와도 자연스럽게 결합 가능

## 기대 효과
- App.tsx 단순화
- screen 전환 로직 명확화
- 실제 제품 아키텍처에 더 가까운 구조 확보
