# voice-architecture-v1.md

## 목표
식단메이트의 핵심 UX인 음성 대화와 핸즈프리 실행을 구현하기 위한 1차 음성 아키텍처.

## 역할 분리
### 1. Whisper / STT
역할:
- 사용자 음성을 텍스트로 변환
- 체크인/명령/수정 요청 인식

적용 구간:
- 대화 모드 체크인
- 식단 교체 요청
- 요리 모드 명령 입력

### 2. Realtime API
역할:
- 실시간 대화형 응답
- 음성 인터랙션 흐름 유지
- 친구 같은 응답 톤

적용 구간:
- 아침/점심 체크인
- 실시간 식단 조정
- 요리 모드 안내

### 3. 계산/플래닝 엔진
역할:
- 대화 결과를 받아 실제 식단/영양/예산 계산 수행
- 음성 모델이 계산을 직접 담당하지 않도록 분리

## MVP 권장 순서
1. STT 우선 연결
2. 체크인/명령 UX 확립
3. 이후 Realtime API로 실시간 대화 강화
4. 요리 모드 핸즈프리 명령 확장

## 보안 원칙
- API 키는 코드에 하드코딩하지 않음
- 환경변수/secret 관리만 허용
- 로컬 개발/배포 secret 분리
