# MEETING_ENGINE_V1.md

## 목적
모든 아이디어/제품/프로젝트 요청을
"지금 무슨 회의를 해야 하는가" 관점에서 처리하기 위한 엔진.

## 기본 템플릿
각 회의는 아래 5개를 항상 포함한다.

1. 회의 목적
2. 이번 회의에서 결정할 것
3. 이번 회의에서 보류할 것
4. 끝나고 남길 산출물
5. 다음 회의 안건

## 회의 출력 형식
- 상태
- 핵심
- 결정
- 보류
- 산출물
- 다음

## 회의 종료 조건
회의가 끝났다고 볼 수 있으려면:
- 최소 1개 이상 명확한 결정이 있어야 함
- 최소 1개 이상 산출물이 있어야 함
- 다음 회의가 정의돼 있어야 함

## Jarvis 규칙
사용자가 짧게 던져도,
Jarvis는 자동으로 회의 프레임을 세운다.

예:
- "이런 거 만들면 어떨까" → 정의 회의
- "이거 돈 되나" → 검증 회의
- "앱 구조 짜보자" → 구조 설계 회의
- "이제 예쁘게 만들어야지" → 디자인 회의
- "바로 만들자" → 구현 회의
