# HANDOFF_PROTOCOL_V1.md

## 목적
agent/session 간 일을 넘길 때,
컨텍스트 손실 없이 실제 회사처럼 인수인계하기 위한 프로토콜.

## 기본 형식
핸드오프는 아래 6개를 포함한다.

1. 배경
2. 현재 단계
3. 이번에 결정된 것
4. 아직 안 결정된 것
5. 상대가 해야 할 일
6. 기대 산출물

## 예시
### research → product
- 배경: 시장 조사 완료
- 현재 단계: 검증 종료 / 구조 설계 시작
- 결정된 것: ICP, 주요 pain point
- 미결정: 가격 정책, MVP 제외범위
- 해야 할 일: 제품 구조 설계
- 산출물: PRD 초안, IA 초안

### product → design
- 배경: 제품 구조 확정
- 현재 단계: 디자인 설계
- 결정된 것: 핵심 화면, 기능 우선순위
- 미결정: 시각 톤, 상태 설계 일부
- 해야 할 일: 화면 흐름 및 설계
- 산출물: 화면 목록, 와이어프레임 문서

## 규칙
- 핸드오프는 짧아도 구조화되어야 함
- “알아서 해줘”는 금지
- 다음 담당자는 받은 문서를 기준으로만 움직임
