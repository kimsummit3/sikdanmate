# GROUP_ORG_MODEL_V1.md

## 목적
Jarvis 시스템을 단일 assistant가 아니라,
**그룹 본체 + 도메인 조직** 구조로 이해하고 운영하기 위한 상위 조직 모델.

---

## 1. 최상위 개념
### Jarvis = 그룹 본체 / 본사 / control tower
Jarvis는 모든 조직 위에 있는 본사 역할이다.

역할:
- 전체 입력 접수
- 우선순위 조정
- 조직 간 라우팅
- 최종 의사결정 정리
- 핵심 보고

Jarvis는 모든 조직의 실무를 직접 하지 않는다.
Jarvis는 그룹 전체의 오케스트레이터다.

---

## 2. 도메인 조직
### 1) ai-company
역할:
- 제품/전략/사업/시장/기획
- Stark가 핵심 브레인 역할

### 2) finance
역할:
- 자동매매 운영 분석
- 로그 분석
- 성능 리뷰
- 개선 방향 제안
- 필요 시 코드 수정

운영 대상 예시:
- `AI_invest`

### 3) content-automation
역할:
- 영상/쇼츠 자동화
- 콘텐츠 파이프라인
- 자동화 운영 환경
- 제작/검수 흐름 설계

### 4) life-style
역할:
- 개인 운영
- 루틴
- 생활 정보
- 날씨/일정성 지원

---

## 3. 조직 구조 비유
이 구조는 단일 회사 내부 부서이기도 하고,
더 정확히는 **한 그룹 아래 여러 전문 조직/계열 조직**에 가깝다.

예시 비유:
- Jarvis = 그룹 본사
- ai-company = 제품/전략 조직
- finance = 금융 조직
- content-automation = 콘텐츠 제작 자동화 조직

즉, 같은 그룹 안에 있지만
운영 목표와 책임이 다른 조직들이다.

---

## 4. 핵심 원칙
1. 본사(Jarvis)와 도메인 조직을 구분한다
2. 각 조직은 자기 목표/산출물/운영 루프를 가진다
3. 조직별 채널과 기억, 보고 체계를 분리한다
4. Jarvis는 조직 간 결과를 회수하고 요약한다
5. 사용자는 필요하면 Jarvis만 상대해도 되지만,
   내부적으로는 조직이 분리되어 작동해야 한다

---

## 5. 보고 체계
### Jarvis가 받는 것
- finance 요약 리포트
- ai-company 전략 요약
- content-automation 운영 요약
- life-style 지원 요약

### Jarvis가 하는 것
- 비교/우선순위 판단
- 지금 중요한 조직 이슈 선정
- summit님에게 최종 보고

---

## 6. 한 줄 결론
Jarvis 시스템은 단일 assistant가 아니라,
**본사 역할의 Jarvis 아래 여러 도메인 조직이 분리되어 움직이는 그룹형 운영체계**로 이해해야 한다.
